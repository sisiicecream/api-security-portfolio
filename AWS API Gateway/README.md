# Lab 06: API Gateway Security Implementation (AWS)

##  Overview
This lab demonstrates securing a serverless API using AWS API Gateway and Lambda.

##  Objective
To implement authentication, rate limiting, and secure access controls in an API environment.

##  Tools Used
- AWS API Gateway
- AWS Lambda
- IAM

##  Approach
- Created Lambda backend API
- Configured API Gateway endpoints
- Implemented Lambda authorizer
- Applied rate limiting and HTTPS enforcement

##  Findings
- Secure APIs require layered controls
- Token-based authentication improves access control
- Rate limiting prevents abuse and DoS attacks

##  Mitigation
- Enforce HTTPS-only communication
- Implement strong authentication mechanisms
- Apply throttling and monitoring
